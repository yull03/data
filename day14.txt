1. 예외(Exception) 처리
(1) 에러(Error)
    컴파일 시 발생하는 에러
    런타임 시 발생하는 에러

(2) 예외
    런타임 시 발생하는 에러 중에
    개발자가 처리 가능한 에러
    cf) 처리가 불가능한 에러
        OOM(Out Of Memory)
            - 예외처리할 메모리 자체가 없음

(3) 예외처리 방법
    try {
        예외가 발생할 가능성이 있는 로직;
    }
    catch(예외명1){
        예외1에 대해 처리할 로직;
    }
    catch(예외명2){
        예외2에 대해 처리할 로직;
    }
    ....
    catch(Exception e){
        예상 불가능한 오류 포함하여 모두에 대해 할 로직;
    }

    * 어쩔 수 없이 예외처리를 해야만 하는 경우
    => 메서드가 throws Exception

(4) 예외 출력 방법
    e.getMessage() : 오류의 기본적인 내용을 문자열로
    e.toString() : 오류의 조금 자세한 내용을 문자열로
    e.printStackTrace() : 오류의 상세 정보를 출력
